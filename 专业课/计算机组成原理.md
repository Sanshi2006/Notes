### 3.12存储器的分类

1.按照介质分类：

①半导体存储器（主存，Cache）：以半导体器件存储信息

②磁表面存储器（磁盘，磁带）：以磁性材料存储信息

③光存储器：以光介质存储信息



2.按存取方式分类：

①随机存取存储器（Random Access Memory，RAM）：读写任何一个存储单元所需时间都相同，与存储单元所在物理位置无关

②顺序存取存储器（Sequential Access Memory，SAM）：读写一个存储单元所需时间取决于存储单元所在的物理位置

③直接存取存储器（Direct Access Memory，DAM）：既有随机存取特性，也有顺序存取特性。先直接选取信息所在区域，然后按顺序方式存取

④相联存储器（Associative Memory）：即可用按内容访问的存储器，可以按照内容检索到存储位置进行读写，“快表”就是一种相联存储器

* ②③都是串行访问存储器：读写某个存储单元所需时间与存储单元的物理位置有关



3.按信息的可更改性分类：

①读写存储器（Read/Write Memory）：即可读、也可写，如磁盘、内存、Cache

②只读存储器（Read Only Memory）：只能读，不能写，BIOS通常写在ROM中



4.按信息的可保存性分类：

①断电后，存储信息消失的存储器--易失性存储器（主存、Cache）

断电后，存储信息依然保持的存储器--非易失性存储器(磁盘、光盘）

②信息读出后，原存储信息被破坏——破坏性读出（如DRAM芯片，读出数据后要进行重写）

信息读出后，原存储信息不被破坏——非破坏性读出（如SRAM芯片、磁盘、光盘）

# 4.指令系统

## 4.1指令的基本格式

1.一条指令通常包括操作码和地址码两部分

操作码（OP）：用于描述用户要做什么

地址码（A）：用于描述对谁进行操作

一条指令可能包括0个、1个、2个、3个、4个地址码

根据地址码数目的不同，可以将指令划分为零地址指令、一地址指令、二地址指令……

### 4.11按地址码数目划分的指令类型

#### ①零地址指令

1.不需要操作数，如空操作、停机、关中断等指令

2.堆栈计算机，两个操作数隐含存放在栈顶和次栈顶，计算结果压回栈顶	

#### ②一地址指令

1.只需要单操作数  如加1，减1，取反，求补等

   指令含义：$OP(A_1)->A1$，完成一次指令需要3次访存：取指->读$A_1$ ->写$A_1$

* $A_1$指某个主存地址   $(A_1)$表示$A_1$所指向的地址中的内容

2.需要两个操作数，但其中一个操作数隐含在某个寄存器（如隐含在ACC）

   指令含义：$(ACC)OP(A_1)->ACC$，完成一次指令需要2次访存：取指->读$A_1$

#### ③二地址指令

|  OP  | $A_1$(目的操作数) | $A_2$(源操作数) |
| :--: | :---------------: | :-------------: |

目的操作数：用于存放操作结果 操作完后会被新值覆盖

源操作数：我们只使用它的值 操作完后不改变

常用于需要两个操作数的算术运算、逻辑运算相关指令

指令含义：$(A_1)OP(A_2)->A_1$

完成一次指令需要访存4次：取指->读$A_1$->读$A_2$->写$A_1$

#### ④三地址指令

|  OP  | $A_1$ | $A_2$ | $A_3$(结果) |
| :--: | :---: | :---: | :---------: |

常用于需要两个操作数的算术运算、逻辑运算相关指令

指令含义：$(A_1)OP(A_2)->A_3$

完成一次指令需要访存4次：取指>读$A_1$->读$A_2$->写$A_3$

#### ⑤四地址指令

| OP   | $A_1$ | $A_2$ | $A_3$（结果） | $A_4$（下址） |
| ---- | ----- | ----- | ------------- | ------------- |

指令含义：$(A_1)OP(A_2)->A_3$

$A_4$为下一条要执行的指令的地址

完成一次指令需要访存4次：取指>读$A_1$->读$A_2$->写$A_3$

### 4.12按指令长度分类指令

概念辨析：

指令字长：一条指令的总长度（可能会变）

机器字长：CPU进行一次整数运算所能处理的二进制数的位数（通常和ALU直接相关 不可变）

存储字长：一个存储单元中的二进制代码位数（通常和MDR位数相同 不可变）

* 半字长指令、单字长指令、双字长指令	——指令长度是机器字长的多少倍

* 指令字长会影响取指令所需时间

* 定长指令字结构：指令系统中的所有指令长度都相等
* 变长指令字结构：指令系统中所有指令的长度不等

### 4.13按操作码长度分类指令

* 定长操作码：指令系统中所有指令的操作码长度都相同  $n$位->$2^n$条指令
* 可变长操作码：指令系统中各指令的操作码长度可变

## 4.2指令寻址

**概念辨析**：

* 指令寻址：下一条要执行的指令的指令地址  始终由程序计数器PC给出
* 数据寻址：确定本条指令的地址码指明的真实地址

①顺序寻址： $(PC)+1->PC$

这里的“1”是指一个指令字长，实际加的值会因指令长度、编制方式的不同而不同

每次取指令后，一定会执行$PC+1$

②跳跃寻址：由转移指令给出（如JMP无条件跳转指令）

## 4.3数据寻址

### 直接寻址

指令中地址码字段给出的地址$A$就是操作数的有效地址$EA(Effective Address)$，即$EA＝A$。

* 取指令访存1次+执行指令访存1次

### 间接寻址

指令的地址字段给出的形式地址不是操作数的真正地址，而是操作数有条地址所在的存储单元的地址，也就是操作数地址的地址，即$EA=(A)$

* 一次间接寻址，取指令访存1次，执行指令需要访存2次，共访存3次

### 寄存器寻址

在指令字中直接给出操作数所在的寄存器编号，即$EA=R_i$，其操作数在$R_i$所指的寄存器内

取指令访存1次+执行指令访存0次

### 寄存器间接寻址

寄存器$R_i$中给出的不是一个操作数，而是操作数所在主存单元的地址，即$EA=(R_i)$

* 取指令访存1次+执行指令访存1次

### 隐含寻址

不显式地给出操作数的地址，而是在指令中隐含操作数的地址

### 立即寻址

一地址指令：

| 操作码（OP） |  #   | 0···011 |
| ------------ | :--: | ------: |

形式地址$A$就是操作数本身，又称为立即数，一般采用补码形式

”#“表示立即寻址特征

* 取指令访存1次+执行指令访存0次

### 基址寻址

将CPU中基址存储器$BR(base\;address\;register)$的内容加上指令格式中的形式地址A，从而形成操作数的有效地址，即$EA=(BR)+A$

* 以程序的起始存放地址作为起点

* 用哪一个寄存器作为基址寄存器也必须在硬件设计时就事先规定，基址寄存器$BR$中的内容称为基准地址，该值可正可负。

### 变址寻址

有效地址$EA$等于指令字中的形式地址$A$与变址寄存器$IX(Index\;Register)$的内容相加之和，即 $EA=(IX)+A$，其中$IX$可为变址寄存器，也可用通用寄存器作为变址寄存器

* 程序员自己决定从里作为起点

* 变址寄存器$IX$中的值称为变址值，该值可正可负

### 相对寻址

把程序计数器$PC$的内容加上指令格式中的形式地址$A$而形成操作数的有效地址，即$EA=(PC)+A$，其中$A$是相对于$PC$所指地址的位移量，该值可正可负，用补码表示

* 以程序计数器$PC$所指地址作为起点

## 5.1CPU的功能和基本结构

### 5.11CPU的功能

1.**指令控制**。完成取指令、分析指令和执行指令的操作，即程序的顺序控制。

2**.操作控制**。一条指令的功能往往是由若干操作信号的组合来实现的。CPU管理并产生由内存去除的每条指令的操作信号，把各种操作信号送往相应的不见，从而实现这些部件按指令的要求进行动作。

3.**时间控制**。对各种操作加以时间上的控制。时间控制要为每条指令按时间顺序提供应有的控制信号。

4.**数据加工**。对数据进行算术和逻辑运算。

5.**中断处理**。对计算机运行过程中出现的异常情况和特殊请求进行处理。

### 5.1.2运算器的基本结构

1.**算术逻辑单元**：主要功能是进行算术/逻辑运算

2.**通用寄存器组**：如$AX、BX、CX、DX、SP$等，用于存放操作数（包括源操作数、目的操作数及中间结果）和各种地址信息等。$SP$是堆栈指针，用于指示栈顶的地址。

3.**暂存寄存器**：用于暂存从主存读来的数据，这个数据不能存放在通用寄存器中，否则会破坏其原有内容。

4.**累加寄存器**：它是一个通用寄存器，用于暂时存放ALU运算的结果信息，用于实现加法运算。

5.**程序状态字寄存器**：保留由算术逻辑运算指令或测试指令的结果而建立的各种状态信息，如溢出标志（OP）、符号表示（SF）、零标志（ZF）、进位标志（CF）等。 PSW中的这些位参与并决定微操作的形成。

6.**移位器**：对运算结果进行移位运算

7.**计数器**：控制乘除运算的操作步数

![image-20251124203612430](https://raw.githubusercontent.com/Sanshi2006/ImagesResources/main/img/image-20251124203612430.png)

### 5.1.3控制器

1.**程序计数器**：用于指出下一条指令在主存中的存放地址。CPU就是根据PC的内容去主存中取指令的。因程序中指令(通常)是顺序执行的，所以PC有自增功能。

2.**指令寄存器**：用于保存当前正在执行的那条指令

3.**指令译码器**：仅对操作码字段进行译码，向控制器提供特定的操作信号

4.**微操作信号发生器**：根据IR的内容(指令)、PSW的内容(状态信息)及时序信号，产生控制整个计算机系统所需的各种控制信号，其结构有组合逻辑型和存储逻辑型两种。

5.**时序系统**：用于产生各种时序信号，它们都是由统一时钟(CLOCK)分频得到

6.**存储器地址寄存器**：用于存放所要访问的主存单元的地址。

7.**存储器数据寄存器**：用于存放向主存写入的信息或从主存中读出的信息

# 6.总线系统

## 6.1基本概念

* 总线宽度：又称为总线位宽，它是总线上同时能传输的数据位数，通常是指数据总线的根数，如32根称为32位（bit）总线

* 总线带宽：可理解为总线的数据传输率，即单位时间内总线上可传输数据的位数，通常用每秒钟传送信息的字节数来衡量，单位可用字节/秒(B/s)表示。

  注：总线带宽是总线本身所能达到的最高传输速率

  总线带宽=总线工作频率×总线宽度（bit/s）=总线工作频率×（总线宽度/8）（B/s）

  总线工作频率和总线周期为倒数关系

## 6.2仲裁方式

### 6.2.1集中仲裁

①链式查询方式：链式查询方式：离中央仲裁器最近的设备具有最高优先权，离总线控制器越远，优先权越低。

* 优点：只用很少几根线就能按一定优先次序实现总线控制，并且这种链式结构很容易扩充设备。
* 缺点：询问链的电路故障很敏感，优先级固定。

②**计数器定时查询方式**：总线上的任一设备要求使用总线时，通过BR线发出总线请求。中央仲裁器接

到请求信号以后，在BS线为“0”的情况下让计数器开始计数，计数值通过一组地址线发向各设备。每

个设备接口都有一个设备地址判别电路，当地址线上的计数值与请求总线的设备地址相一致时，该设

备 置“1”BS线，获得了总线使用权，此时中止计数查询。 每次计数可以从“0”开始，也可以从中止点开

发始。如果从“0”开始，各设备的优先次序与链式查询法相同，优先级的顺序是固定的。如果从中止点

开始，则每个设备使用总线的优级相等。

* 优点：1°计数初始值可以改变优先次序

  ​	    2°对电路的故障没有链式敏感

* 缺点：1°增加了控制线数：若设备有$n$个，则需要$⌈log_{2}{n}+2⌉$条控制线

  ​	    2°控制相对比链式查询相对复杂

③**独立请求方式**：![image-20251129151354444](https://raw.githubusercontent.com/Sanshi2006/ImagesResources/main/img/image-20251129151354444.png)

* 优点：1°响应速度快，总线允许信号BG直接从控制器发送到有关设备，不必再设备间传递或者查询

​                  2°对优先次序的控制相当灵活

* 缺点：1°控制线数量多：若设备有$n$个，则需要$2n+1$条控制线，其中$+1$为$BS$线，用于设备想着总线控制部件反馈是否正在使用总线

​                  2°总线的控制逻辑更加复杂

### 6.2.2分布仲裁

特点：不需要中央仲裁器，每个潜在的主模块都有自己的仲裁器和仲裁号，多个仲裁器竞争使用总线

当设备有总线请求时，它们就把各自唯一的仲裁号发送到共享的仲裁总线上;每个仲裁器将从仲裁总线

上得到的仲裁号与自己的仲裁号进行比较;如果仲裁总线上的号优先级高，则它的总线请求不予响应，

并撤销它的仲裁号最后，获胜者的仲裁号保留在仲裁总线上。

### 6.3.2异步定时

## 7.1磁盘存取时间的三部分

磁盘存取一个数据的总时间=平均寻道时间+平均等待时间+数据传输时间

①平均寻道时间：1°直接给出

2°$平均寻道时间=\frac{最大寻道时间}{2}$（最大寻道时间为磁头从最内圈到最外圈的时间）

②平均等待时间

$平均等待时间=\frac{1}{2×转速（转/秒）}=\frac{60}{2×转速（转/分）}$

③数据传输时间

$数据传输时间=\frac{数据块大小}{磁道容量×转速（转/秒）}$

## 7.2格式化容量计算

①容量（字节）=有效面数 × 每面磁道数 × 每磁道扇区数 × 每扇区字节数

有效面数需排除最上和最下的无用面

②$容量（字节）=\frac{总磁道数×单磁道容量（位）}{8}$

总磁道数=道密度 × 内外半径差

单磁道容量=位密度 × 磁道周长

## 7.3显示系统

# 8.输入输出系统

## 8.1极高频考点

### 8.1.1四种$I/O$控制方式对比

| 对比维度     | 程序查询方式                                       | 程序中断方式                                    | DMA方式                                                      | 通道方式                                                     |
| ------------ | -------------------------------------------------- | ----------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| CPU介入程度  | 全程介入（循环查询设备状态，忙等待）               | 仅在中断响应、保存/恢复现场、执行服务程序时接入 | 仅预处理（初始化DMA控制器：设置内存地址、传输长度）和后处理（接收中断）介入 | 几乎不介入（仅发送启动通道指令，通道自主执行通道程序）       |
| 数据交换单位 | 字（字节）                                         | 字（字节）                                      | 数据块（批量传输）                                           | 数据块（支持多设备并行传输）                                 |
| 硬件成本     | 低（仅需设备选择电路、数据缓冲寄存器，状态寄存器） | 中（需中断控制器、中断请求/允许/屏蔽触发器）    | 中高（需专用DMA控制器：内存地址计数器、字计数器、数据缓存寄存器） | 高（需独立$I/O$处理机，含指令寄存器，程序计数器）            |
| CPU效率      | 最低（查询期间CPU无法执行其他程序）                | 中（CPU可执行主程序，仅设备就绪时中断）         | 高（数据传输全程无需CPU干预，CPU可执行其他任务）             | 最高（可同时管理多个设备并行传输，完全解放CPU）              |
| 适用场景     | 简单低速设备（如温度传感器，简单打印机）           | 中低速设备（如键盘、鼠标、普通打印机）          | 高速大批量数据设备（如磁盘、磁带、网卡）                     | 大型计算机系统、多高速设备（如磁盘阵列、大型服务器$I/O$子系统） |

### 8.1.2DMA方式核心机制

* **数据通路本质**：DMA 建立的是**主存与外设**之间的直接数据通路，数据不流经 CPU（避坑：易误选 “CPU 与外设” 或 “CPU 与主存”，需区分 “控制通路” 与 “数据通路”——CPU 仅通过指令控制 DMA 控制器，不参与数据传输）。

* **CPU干预边界**：仅在 “预处理” 和 “后处理” 阶段介入，传输过程完全由 DMA 控制器控制。

  预处理：CPU 设置 DMA 控制器的内存起始地址、传输数据量、传输方向（读 / 写）。

  后处理：DMA 传输完成后发中断，CPU 执行中断服务程序（如数据校验、释放缓冲区）。

* 三种传输方式：

| 传送方式    | 特点                                                         | 适用场景                                             |
| ----------- | ------------------------------------------------------------ | ---------------------------------------------------- |
| 停止CPU访内 | DMA占用总线直到批量传输完成，控制简单，CPU暂停工作           | 高速设备成组传送（如磁带机）                         |
| 周期挪用    | DMA “窃取” CPU 的总线周期传输数据，CPU 仅暂停一个周期，不影响主程序连续性 | 中高速设备（如磁盘）                                 |
| 交替访内    | CPU 与 DMA 按固定时钟交替使用总线，并行工作，又称 “透明 DMA”，硬件逻辑复杂 | 高速且需 CPU 持续工作的场景（如 GPU 与内存数据交换） |

## 8.2高频考点

### 8.2.1程序中断方式处理流程

1.**中断请求**：外设完成操作后，向中断控制器发送中断请求信号，置位中断请求触发器（IR=1）。

关键：仅当设备允许中断（EI=1）且未被屏蔽（IM=0）时，请求才会被提交给 CPU。

2.**中断响应**：CPU 执行完当前指令后，检测中断请求（若 IF=1，开中断状态），关中断（防止嵌套混乱），保存断点（PC 值压栈）和 CPU 现场（寄存器状态）。

关键：中断响应时机是 “当前指令执行完毕”，而非指令执行中（异常在指令执行中检测）。

3.**中断处理**：根据中断号查找中断向量表，获取中断服务程序入口地址，转至服务程序执行（如数据读写、错误处理）。

关键：中断向量表的作用是存储 “中断服务程序入口地址”，而非设备地址或断点地址。

4.**中断返回**：恢复 CPU 现场（从栈中弹出寄存器状态和 PC 值），开中断（允许后续中断），返回主程序继续执行。

### 8.2.2中断分级与屏蔽逻辑

1.**中断优先级**：系统按设备重要性 / 速度设定优先级，高级中断可中断低级中断服务程序，同级中断按排队链或独立请求方式判别。

示例：二维中断系统中，2 级设备（如磁盘）优先级高于 1 级（如打印机），1 级高于 0 级（如键盘）。

2.**中断屏蔽（IM）**：CPU 执行某级中断服务程序时，置位该级及低级的 IM 位（屏蔽），仅开放高级 IM 位（允许高级中断嵌套）。

示例：执行 1 级设备（如 D）的中断服务程序时，IM2=0（开放 2 级）、IM1=1（屏蔽 1 级）、IM0=1（屏蔽 0 级）。

3.**单独屏蔽某设备**：通过置位该设备的 “中断允许触发器（EI=0）” 实现，而非修改 IM（IM 屏蔽整级设备）。

## 8.3中频考点

### 8.3.1$I/O$接口功能与编址

* 五大功能：

| 功能               | 作用                                          |
| ------------------ | --------------------------------------------- |
| 地址译码与设备选择 | 解析 CPU 发出的 I/O 地址，选中目标设备        |
| 数据缓冲           | 暂存 CPU 与外设间的传输数据，解决速度匹配问题 |
| 控制与定时         | 生成同步信号，协调 CPU 与外设的传输时序       |
| 信号格式转换       | 实现串行 / 并行转换（如 USB 接口）、电平转换  |
| 错误检测           | 通过奇偶校验、CRC 校验检测传输错误            |

* 两种编址方式：

| 编制方式                   | 特点                                                     | 适用场景                                 |
| -------------------------- | -------------------------------------------------------- | ---------------------------------------- |
| 统一编址（存储器映射 I/O） | 将 I/O 端口当作内存单元，无专用 I/O 指令，用访存指令访问 | CPU 无专用 I/O 指令的系统（如 ARM 架构） |
| 独立编址（I/O 映射 I/O）   | I/O 端口有独立地址空间，需专用 I/O 指令（如 IN/OUT）     | 有专用 I/O 指令的系统（如 x86 架构）     |

### 8.3.2通道方式核心特性

* **与 DMA 的差异**：通道有独立指令系统（通道程序），可管理多个设备并行传输；DMA 仅能控制单个设备的批量传输。

* 三种通道类型：

| 通道类型     | 特点                                                         | 适用场景                     |
| ------------ | ------------------------------------------------------------ | ---------------------------- |
| 选择通道     | 每次仅服务一个设备，成组传输，速率高                         | 高速单设备（如磁盘）         |
| 字节多路通道 | 分时服务多个低速设备，按字节传输                             | 多低速设备（如键盘、打印机） |
| 数组多路通道 | 结合选择通道（成组传输）和字节多路通道（分时），服务多个高速设备 | 多高速设备（如磁盘阵列）     |

## 8.4低频考点

### 8.4.1总线定时方式

| 定时方式 | 特点                                                      | 适用场景                                        |
| -------- | --------------------------------------------------------- | ----------------------------------------------- |
| 同步定时 | 统一时钟信号，总线周期固定，控制简单                      | 总线长度短、设备存取时间接近（如 CPU 与 Cache） |
| 异步定时 | 无统一时钟，通过握手信号（请求 / 应答）同步，总线周期可变 | 设备存取时间差异大（如 CPU 与打印机）           |

### 8.4.2通用$I/O$接口标准

| 接口标准 | 传输方式 | 特点                                                     | 适用场景                       |
| -------- | -------- | -------------------------------------------------------- | ------------------------------ |
| SCSI     | 并行     | 高速智能，可连接 1-15 台设备（磁盘、光盘等），硬件成本高 | 服务器、工作站                 |
| IEEE1394 | 串行     | 高速实时，设备平等（无需 PC 介入自成系统），体积小       | 家电（如数码相机）、计算机外设 |
