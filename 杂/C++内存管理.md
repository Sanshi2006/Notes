

# 一.析构函数

析构函数是C++类中一个特殊的函数，名字和类名相同，不过前面多了一个`~`

在位运算中，`~`表示为按位取反，这里可以理解为“构造函数的反面”

它的作用是，在对象生命周期结束时，清理对象占用的资源

* 析构函数一般不需要自己显示地定义，在创建类时会默认创建一个

但是如果你在类中有如下操作，那么应该显式地自定义析构函数：

比如如果类中`new`了一个数组，那么析构函数就要`delete`掉这个数组

如果在类中打开了文件，那么在析构函数就要执行`fclose`或者`close`操作

如果在类中连接了数据库，那么在析构函数中就要断开



* 例：

```cpp
#include<bits/stdc++.h>

struct dat{
    dat() = default;        
    ~dat(){
        std::cout << "done\n";
    }
};
int main(){

    dat d = dat();

    return 0;
}
```

在上述代码中，我们显示地定义了dat类的析构函数，执行后你会发现输出了done，说明在对象生命周期结束后会自动调用析构函数



# 二.`unqiue_ptr`

## 1. `unique_ptr`是什么

`unique_ptr`是C++中的智能指针，它独享对某一内存区域的所有权

* 同一时刻，只能有一个`unique_ptr`指向特定的堆内存空间

* 无法复制，但可以通过`move`函数转移所有权

* 当`unique_ptr`被销毁时，无需手动操作，会自动释放内存

  

## 2. `unique_ptr`的定义方式

使用`std::unique_ptr<类名> 变量名 = std::make_unique<类名>();` 创建



# 三. `shared_ptr`

## 1. `shared_ptr`是什么

`shared_ptr`同样是C++中的智能指针，但是它允许同一时刻有多个指针指向同一内存区域

* 引用计数：`shared_ptr`内部维护了一个计数器，每多一个独立的`shared_ptr`指向这个对象时，计数器就回+1（注意如果是引用的话不增加计数）
* 自动释放：当计数器值为0，即没有一个`shared_ptr`指向它时，它就回自动释放内存
* 允许复制



## 2.`shared_ptr`的定义方式

使用`std::shared_ptr<类名> 变量名 = std::make_shared<类名>();` 创建



# 四.RAII

即**Resource Acquisition Is Initialization**（资源获取即初始化）

* 栈上的对象在超过作用域后，一定会调用析构函数释放空间，可以认为栈上的对象只是一个中介，通过它，析构函数可以释放在堆上产生的内存空间

优势：

* 杜绝资源泄漏
* 出现异常时，栈上已经创建的对象仍然会执行析构函数，释放空间
